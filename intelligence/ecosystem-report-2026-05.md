### Why Do Servers Fail?

All 26 failures returned the same error: `MCP error -32000: Connection closed`. Root causes include:

1. **Missing API keys** — Many servers require environment variables (e.g., `GITHUB_TOKEN`, `BRAVE_API_KEY`) that aren't set
2. **Broken packages** — Published to npm but never tested as an MCP server
3. **Dependency issues** — Packages that fail to install cleanly via `npx -y`
4. **Unmaintained** — Published months ago, dependencies have drifted

## Implications

1. **For agent developers**: Don't assume MCP servers "just work." Over half fail silently. Always implement fallbacks (see [agentvault-retry](https://www.npmjs.com/package/agentvault-retry)).

2. **For server authors**: Test your package with `npx -y your-package` before publishing. Many servers work in dev but fail in production.

3. **For the ecosystem**: We need quality standards. [AgentScore](https://www.npmjs.com/package/agentvault-score) provides automated quality scoring — scan any server with `npx agentvault-score scan "your-target"`.
